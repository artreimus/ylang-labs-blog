## Claim Boundaries

- Treat Seedance 2.0 layer counts, parameter counts, FLOPs, optimizer settings, exact attention design, dataset inventory, hardware, and total training compute as undisclosed unless ByteDance publishes a fuller technical report.
- Treat the diffusion-transformer lineage as an inference from Seedance 1.0 and 1.5 Pro public disclosures, not as a confirmed 2.0 implementation detail.
- Treat Dola Seed as a naming/product-family context unless a primary ByteDance source publishes a more precise implementation dependency.
- Treat ByteDance benchmark rankings as official/public claims unless independently reproduced with a separate evaluation suite.
